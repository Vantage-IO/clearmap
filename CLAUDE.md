# CLAUDE.md: ClearMap

ClearMap: a local-first HIPAA **technical-risk** scanner/guardrail for healthcare codebases. Three layers: deterministic (Semgrep rules in `rules/` + Gitleaks via `scripts/scan.py`), LLM reasoning (checklists in `references/`), and scoring (`scripts/scoring.py`) producing a markdown/HTML report. It ships as an agent plugin (`.claude-plugin/` + `.codex-plugin/` + `.agents/`, `commands/` = `/clearmap:setup|plan|develop|audit|issues|report`, `skills/` = clearmap-development + clearmap-audit). Apache 2.0. Core is Python 3.10+ stdlib only: no runtime pip dependencies.

**It is a technical risk signal, NOT a compliance certification. Never emit "HIPAA compliant" language in any output (enforced by the banned-phrase guard in `report.check_banned`).**

## Scoring invariants (hard rules)

- Never invent or "randomly calculate" scores. Every number must trace to findings. A blended score requires an actually-run reasoning pass; never score a fixture's answer key.
- The topline score is the **blended** composite and errs on the side of caution.
- The score can **never be 100** when any issue exists. ≥1 critical caps it in the 75s; multiple criticals lower it further (saturating decay: floor 40, decay 0.55). No criticals: ceilings 95/90/85 (none/medium/high).
- Non-applicable categories are shown as **N/A**, never scored 100. Weights renormalize across applicable categories.
- **Reasoning-only categories** (AI-RAG, AUDIT: no deterministic rules exist for them) must NEVER be scored 100 on a deterministic-only run. When `source_layer` lacks "reasoning" and they apply with no findings, scoring marks them `not_reviewed` (excluded from the composite) and the report shows an "Assessment incomplete" banner + qualifies the score "(automated layer only)". The principle is honesty: an incomplete assessment must be labeled as incomplete, never presented as a clean result.
- Category names and findings map to real HIPAA Security Rule citations (45 CFR 164.312) via the versioned `references/regulatory-baseline.json`; every `hipaa_ref` must resolve there.

## Report architecture & style

- `report.build_model()` decides ALL report content once; `render_md()` and `report_html.render_html()` are mechanical emitters. Content changes go in the model, never in one emitter.
- Professional prose: no raw rule slugs as headings, no enum tokens ("deterministic"/"reasoning"), no underscores or "codey looking things" in user-facing output. The words are "finding", "Confirmed", "Needs verification".
- **No em or en dashes in rendered output** (`_normalize_dashes` enforces; an invariant test checks).
- Executive summary first; scoring methodology in Appendix A; disclaimer + closing note in Appendix B. No price in any report.
- Always print the output file path. Audit outputs go to `.clearmap/` in the target repo (gitignored, fixed filenames).

## Regeneration rules

- `examples/*/expected-findings.json` are GENERATED by `examples/build_manifest.py`. Never hand-edit; CI diffs them.
- Golden fixtures (`tests/fixtures/golden/report.md` + `.html`) are byte-pinned. On intentional content change regenerate with `python3 scripts/report.py tests/fixtures/golden/findings.json --repo golden-fixture --date 2026-01-01 --format both --out tests/fixtures/golden/report.md` in a code-free commit and review that diff line by line.

## Testing & validation

- Engine pins: `semgrep` 1.164.x, `gitleaks` 8.30.x (`init.py doctor` checks; CI installs exactly these).
- `python3 -m unittest discover -s tests -v` (PHI-leak canary + golden reports included). Lint: `ruff check scripts tests` (config in pyproject.toml).
- Byte-stable scan output is an invariant: diff two consecutive runs.
- Calibration gates: deterministic recall ≥0.94 (measured baseline 0.95), precision 1.0, safe fixture 0 findings. When detection coverage improves, raise the gate; never lower it.
- Validate rules with `semgrep --validate --config rules/`.
- Semgrep gotcha: `metavariable-regex` is anchored (`re.match`); patterns usually need a leading `.*`.

## Domain notes

- Encryption in transit: external traffic must be TLS; internal service-to-service behind a VPC may legitimately use HTTP, so don't flag it as critical without context (see `interpretation_notes` in the regulatory baseline).
